Mars Rover

(Assignment #3)

The Mars Rover program, written in Javascript using Node.js on replit.com, is a simulation for issuing commands to a Mars rover from mission control. Each list of commands is converted into a message and sent to the rover. Upon receiving the message, the rover carries out the commands and creates a response with a completion status for each command. This project was creating using the Test Driven Development software development process whereby each unit test is written (and run) before the corresponding code for each discrete feature is written. The code can be refactored (also done in a TDD process) and the same tests can be used for regression testing.

Concepts learned:

  • Unit Testing using Jasmine, a popular behavior-driven development framework for testing JavaScript code
  • Using the describe() function for a suite of tests
  • Using the it() function for individual test specifications
  • Expectations with the expect() function, which takes a value, called the actual, for comparison to the expected value
  • The expectation function is chained with a built-in Matcher function, which compares the expected value to the actual value
  • Test Driven Development - a software development process where each unit test is written first before writing the corresponding code
  • Exporting functions from modules
  • Classes as templates for creating objects - with class constructors and methods
  • Creating and using instances of objects from classes

HTML Me Something

(Assignment # 4)

This project uses HTML and CSS to build a web page. This project involved local development with Visual Studio Code and Git, with changes pushed to a GitHub repository. GitHub Pages, a static site hosting service, was used to pull the HTML and CSS files straight from the project repository on GitHub and publish to a GitHub Pages project site.

Concepts learned:

  • HTML for building a web page from scratch
  • CSS to separate content from design and layout
  • Remote repositories with GitHub and GitHub Pages
  • Local development with VS Code and Git Bash
  • Git commands learned included:
    • Getting and Creating Projects – init, clone
    • Basic snapshotting – add, status, commit, diff, reset, rm
    • Branching and merging – branch, checkout, merge, stash
    • Sharing and updating projects – fetch, pull, push, remote
    • Inspection and Comparison – diff, log, show
