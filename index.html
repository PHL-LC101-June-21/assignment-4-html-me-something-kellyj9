Mars Rover

(Assignment #3)

The Mars Rover program, written in Javascript using Node.js on replit.com, is a simulation for issuing commands to a Mars rover from mission control. Each list of commands is converted into a message and sent to the rover. Upon receiving the message, the rover carries out the commands and creates a response with a completion status for each command. This project was creating using the Test Driven Development software development process whereby each unit test is written (and run) before the corresponding code for each discrete feature is written (and tested). The code can be refactored (also done in a TDD process) and the same tests can be used for regression testing.

Concepts learned:

  • Unit Testing using Jasmine (Jasmine is a popular behavior-driven development framework for testing JavaScript code).
  • using the describe() function for a suite of tests
  • using the it() function for individual test specifications
  • Expectations with the expect() function, which takes a value, called the actual
  • The expectation function is chained with a built-in Matcher function, which takes the expected value
  • Test Driven Development - a software development process where each unit test is written first before writing the corresponding code
  • Exporting functions from modules
  • Classes – constructors and class methods
  • Creating instances of objects from classes

HTML Me Something

(Assignment # 4)

This project uses HTML and CSS to build a web page. This project involved local development with Visual Studio Code and Git, with changes pushed to a GitHub repository. GitHub Pages, a static site hosting service, was used to pull the HTML and CSS files straight from the project repository on GitHub and publish to a GitHub Pages project site. The URL for this web page is: kellyj9.github.io/html-me-something

Concepts learned:

  • HTML
  • CSS
  • Remote repositories with GitHub and GitHub Pages
  • Local development with VS Code and Git Bash
  • Git commands learned included:
    • Getting and Creating Projects – init, clone
    • Basic snapshotting – add, status, commit, diff, reset, rm
    • Branching and merging – branch, checkout, merge, stash
    • Sharing and updating projects – fetch, pull, push, remote
    • Inspection and Comparison – diff, log, show
